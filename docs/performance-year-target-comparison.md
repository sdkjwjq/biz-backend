## 1. 结论摘要

年度绩效目标数据与客户原始 Excel **明显不一致**，不是个别录入差异。

处理决议：

- A 类、B 类、C 类年度目标问题已纳入迁移脚本 `data/migrations/2026-05-30-performance-relation-year-fix.sql`。
- 迁移后按 Excel 重建 2025-2029 年度目标：缺失记录补齐，已有记录覆盖为 Excel 年度目标。
- D 类文本型建设期目标保持现状，不新增字段；文本类继续用现有数字映射，`1` 或 `100`。
- 该迁移脚本已在本地库执行验证：年度目标缺失 `0`、目标值不一致 `0`、数据库额外年度记录 `0`。

- 年度目标建议以客户 Excel 为准重建。
- 文本型目标需要先确认保存方式，不能直接硬塞到 decimal 字段。

## 9. 建议处理方案

### 9.1 年度目标值以 Excel 重建

建议基于 Excel 重新生成 `biz_performance_year`：

1. 按 `perf_code` 找到 `perf_id`。
2. 对每个绩效生成 2025-2029 五条年度记录。
3. 已存在年度记录则更新 `target_value`。
4. 缺失年度记录则新增。
5. `actual_value` 是否保留，需要在执行前确认；建议先做快照。

### 9.2 文本型目标先不要硬塞进 decimal

短期可以继续用现有数字映射参与流程，但建议新增 `target_text` 或类似字段保留 Excel 原文。

### 9.3 修复前必须做快照

年度绩效目标属于基础配置数据。迁移脚本应先备份 `biz_performance_year`、`biz_performance.target_value`，再执行覆盖。
