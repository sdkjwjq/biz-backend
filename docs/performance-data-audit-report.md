## 处理建议

当前代码已做防御：年度错配、已删除对象、`task.data_type = 0` 的关联在绩效刷新时不会计入。

数据层后续可选修复方式：

1. 年度错配：人工确认后，将 `rel_task_performance.year_id` 改到任务 `phase` 对应的 `biz_performance_year.year_id`，或删除错误关联。
2. `data_type = 0` 关联：若确认为“无绩效影响”，建议从 `rel_task_performance` 删除；若实际应影响绩效，则改任务 `data_type`。
