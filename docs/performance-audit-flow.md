# 绩效审核流程说明

## 流程节点

手动填报绩效采用两级处理：

1. 绩效填报：`leader_id` 或管理员提交填报值，生成绩效审核单。
2. 专业群审核：`auditor_id` 处理，审核通过后流转到管理员。
3. 完结归档：管理员处理；这里既是管理员审核，也是最终归档，不再拆成两个业务节点。

## 状态约定

- `10`：专业群审核中
- `20`：待完结归档
- `30`：已完结归档
- `-10`：专业群退回
- `-20`：归档退回
- `0`：已撤回

## 通知来源类型

`sys_notice.source_type` 是短码字段：

- `0`：任务相关通知
- `1`：旧业务/其他通知
- `2`：绩效审核单通知，`source_id` 对应 `biz_performance_submission.sub_id`

绩效待审通知应跳转审核中心，并通过 `subId` 定位绩效审核单。
